# 2026-04-11 - OpenPool Hybrid Network & Browser Swarm

## Phase 1: Legacy Security & Protocol Fixes (April 9)
- **RCE Prevention:** Removed Python exec execution in `node.go`, replaced with hard error
- **Database Hardening:** Added WAL mode and IMMEDIATE transactions in `ledger.go`
- **Auth Security:** Implemented bcrypt hashing for API keys in `auth.go`
- **Float Verification:** Replaced SHA-256 hashing with relative error tolerance (ε=1e-5) in `verifier.go`
- **Dual-Dispatch:** Updated `scheduler.go` to send tasks to 2+ independent peers
- **Staking:** Added `staked_credits` column and collateral requirements in `ledger.go`

## Phase 2: Hybrid Networking (LAN/WAN) - April 11
- **mDNS LAN Discovery:** Implemented local peer discovery via multicast DNS
- **Kademlia DHT:** Global WAN discovery integration
- **Peer Tagging:**
  - LAN peers: `network_type: LAN` (high priority)
  - WAN peers: `network_type: WAN` (low priority)
  - Browser peers: `client_type: browser`
  - Native peers: `client_type: native`
- **Transport:** Added WebSocket support for browser connectivity

## Phase 3: Scheduler Spillover & Hardware Awareness
- **Compute Strategy:** Added `StrategyHybridAuto`, `StrategyLANOnly`, `StrategyWANOnly`
- **Hardware Requirements:** 
  - `HardwareRequirement: Tier1_Native` - GPU/HPC workloads
  - `HardwareRequirement: Tier2_Browser` - Lightweight WASM tasks
- **LAN-First Routing:** Scheduler drains local peers before spilling to WAN
- **Hardware Filtering:** Prevents Tier1_Native tasks from routing to browser nodes

## Phase 4: Browser Swarm Client
- **WASM Worker:** `/ui/wasm-worker.js` - Offloads compute to Web Worker
- **Browser Node:** `/ui/browser-node.js` - js-libp2p client for browser swarm
- **Connection:** Dials native node via WebSocket transport

## Phase 5: Security Hardening
- **Connection Limits:** Added ConnectionManager (100-1000 connections, 30s grace)
- **Client Type Tagging:** Browser vs native peer classification
- **No Hard Kill Switches:** Both mDNS and DHT run concurrently
